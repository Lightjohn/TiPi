    public abstract class WeightedConvolutionCost
extends java.lang.Object
implements DifferentiableCostFunction
    Implements a quadratic cost function for models by FFT-based convolution.

    Definition

    The ``weighted convolution'' cost writes:

    f(x) = (H.x - y)t.W.(H.x - y),

    with H a convolution operator and W a diagonal positive semi-definite weighting operator. The convolution operator writes:

    H = R.F*.diag(F.h).F

    with F the FFT (Fast Fourier Transform) operator, h the point spread function (PSF) and R a linear operator which selects a sub-region of the output of the convolution. The * superscript denotes the adjoint of the operator (complex transpose in this specific case) and diag(v) is a diagonal operator whose diagonal elements are those of the vector v.

    Currently only diagonal weighting operators are implemented, thus:

    W = diag(w) .

    If this restriction is inappropriate for your purpose, you may build your own QuadraticCost function.

    Usage in deconvolution problems

    Assuming uncorrelated Gaussian noise and that the cyclic convolution (with sufficient zero padding) is suitable to approximate the effects of the instrument, the cost function f(x) can be used to implements the likelihood of the data y given the parameters y and with W = diag(w) the precision matrix of the noise (also known as the statistical weights of the data).

    Synopsis

    Because of the many different possibilities (1D, 2D or 3D arrays with float or double elements), creating a new instance of the cost function involves a factory. A typical usage of the code is: 

     // Create operator:
 WeightedConvolutionCost cost = WeightedConvolutionCost.build(objectSpace, dataSpace);

 // Specify the PSF (mandatory):
 cost.setPSF(h);

 // Specify the weights and the data (y mandatory, w can be null):
 cost.setWeightsAndData(w, y);
    where the arguments are:
    • The vector space objectSpace of the variables x (the object).
    • The vector space dataSpace of the variables y (the data). The data space may be smaller than the object space which involves zero padding (symbolically implemented by the operator R). In our implementation, R is defined by the dimensions of the data space and the position of the first element to select in the result of the cyclic convolution H (thus we only consider rectangular output spaces). By default, the data space (if smaller than the result of the convolution) is taken to be approximately the central part of the output of H. It is possible to specify a different position by calling the build method with a list of offsets as additional argument.
    • The PSF h is a shaped vector of the object space or a shaped array (in the former case, the PSF must be appropriately centered, in the sense of the FFT; in the latter case, the shaped array is zero-padded and rolled for you; if the center of the PSF is not at the geometric center of the PSF array, you may specify its position).
    • The data y is a shaped vector of the data space or a shaped array with the same dimensions as the those of the data space.
    • The weights w can be unspecified (with w = null) which is the same as having all weights equal to 1; otherwise w must be the same kind of object as y (shaped array or shaped vector with the same dimensions) and all weights must be nonnegative. If weights are specified, the data are never used where the weights are equal to zero which is a consistent way to indicate missing data.

      • build

        public static WeightedConvolutionCost build(ShapedVectorSpace objectSpace,
                            ShapedVectorSpace dataSpace)
        Build a weighted convolution cost function with centered data.

        The object space of the operator corresponds to the variables denoted as x in WeightedConvolutionCost; while the data space corresponds to the variables denoted as y in WeightedConvolutionCost.

        The size of the data space must be smaller or equal that of the object space which is also the input and output spaces of the cyclic convolution. The rank of the object and data spaces must be the same and comparing their "sizes" involves a comparison for all dimensions. If the shape of the data space is smaller than that of the object space, then the central part of the result of the cyclic convolution is considered as the region corresponding to the data. The factory build(ShapedVectorSpace, ShapedVectorSpace, int[]) can be used to select a different region.

        The returned object is not a valid cost function until you set the point spread function (PSF) with one of the #setPSF() methods and the data and, optionally, the weights with one of the #setWeightsAndData() methods.

        See WeightedConvolutionCost for a detailed description of what is computed by this cost function.

        Parameters:
        objectSpace - - The object space.
        dataSpace - - The data space.
        Returns:
        An instance of the weighted convolution cost function.
        See Also:
        #build(ShapedVectorSpace, ShapedVectorSpace, int[])}

      • build

        public static WeightedConvolutionCost build(ShapedVectorSpace objectSpace,
                            ShapedVectorSpace dataSpace,
                            int[] dataOffset)
        Build a weighted convolution cost function.

        This version of the factory to build a weighted convolution cost function let you specify precisely the position of the region corresponding to the data in the result of the convolution. The offsets of this region must be such that: 

         0 <= dataOffset[k] <= objectDim[k] - dataDim[k]
        where objectDim and dataDim are the respective dimensions of the object and data spaces. If this does not hold (for all k), an ArrayIndexOutOfBoundsException is thrown.

        See build(ShapedVectorSpace, ShapedVectorSpace) for more details on the meaning of the object and data spaces and WeightedConvolutionCost for a more general overview.

        Parameters:
        objectSpace - - The object space.
        dataSpace - - The data space.
        dataOffset - - The relative position of the data within the output of the convolution. It must have as many values as the rank of the object and data spaces of the operator.
        Returns:
        A weighted convolution cost function.
        See Also:
        #build(ShapedVectorSpace, ShapedVectorSpace)}

      • computeCostAndGradient

        public double computeCostAndGradient(double alpha,
                            Vector x,
                            Vector gx,
                            boolean clr)
        Description copied from interface: DifferentiableCostFunction
        Compute the value of the cost function and its gradient.

        This method is intended to compute the value and the gradient of a cost function which are needed by limited memory non-linear optimization methods like the non-linear conjugate gradient and quasi-Newton methods such as L-BFGS.

        The rationale of the alpha and clear arguments is to let one builds a composite cost function efficiently. Below is the pseudo-code of an example which shows how to compute the value and the gradient of a function which is a linear combination of other functions (in the array f) with different multipliers (in the array alpha): 

             DifferentiableCostFunction[] f;
     double[] alpha;
     Vector x;
     Vector gx;
     double fx;
     fx = 0.0;
     for (int j = 0; j < f.length; ++j) {
         fx += f[j].computeCostAndGradient(alpha[j], x, gx, (j == 0));
     }
        Note that the gradient vector has to be cleared for the first (j == 0) function of the list.
        Specified by:
        computeCostAndGradient in interface DifferentiableCostFunction
        Parameters:
        alpha - - A non-negative multiplier for the cost.
        x - - The vector of variables.
        gx - - The vector to store the gradient of the cost function times the weight.
        clr - - Indicate whether the gradient vector has to be cleared (that is, filled with zero) prior to the computation. If false, the contents of the gradient vector is incremented with the gradient of the cost function.
        Returns:
        The value of the cost function times the weight.
