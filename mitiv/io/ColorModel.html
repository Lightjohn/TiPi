Enum ColorModel

    public enum ColorModel
extends java.lang.Enum<ColorModel>
    Different color models used when interpreting shaped arrays as images or when selecting different color channels or applying color conversion.

    Conventions

    A simple heuristic is applied to determine the color model of a shaped array when it is interpreted as an image. The rules are:

    1. a 2D shaped array of any type is a gray-scale image with color model GRAY;
    2. a 3D shaped array of any type with first dimension equals to 3 is a RGB image with color model RGB;
    3. a 3D shaped array of type byte and with first dimension equals to 4 and is a RGBA image with color model RGBA;
    4. anything else corresponds to the the color model NONE.

    These conventions are assumed by the various conversion routines, e.g. DataFormat.makeBufferedImage(mitiv.array.ShapedArray) to convert a buffered image to a shaped array or DataFormat.imageToShapedArray(java.awt.image.BufferedImage) to perform the opposite conversion.

    When reading data as a shaped array, the method guessColorModel(mitiv.array.ShapedArray) can be used to determine the color model following the aforementioned rules.

    The methods filterImageAsFloat(mitiv.array.ShapedArray, mitiv.io.ColorModel) and filterImageAsDouble(mitiv.array.ShapedArray, mitiv.io.ColorModel) may be used to filter or convert an image represented by a shaped array into a specific color model with a floating-point representaion. These methods can also be used to extract a specific color channel or the alpha channel.

    Gray-scale images

    A gray-scale image is stored into a shaped array as a 2D array with the first dimension equals to the width of the image and the second dimension equals to the height of the image. The pixels of a gray-scale image have a single value: their intensity or their gray level. The gray level of a shaped array interpreted as a gray-scale image can be accessed as follows: 

          gray = img.get(x,y);
    where img is an Array2D object and (x,y) are the pixel coordinates.

    RGB images

    A RGB image is stored into a shaped array as a 3D array with the first dimension equals to 3, the second dimension equals to the width of the image and the third dimension equals to the height of the image. The pixels of a RGB image have three colors: red (R), blue (B) and green (G). The colors of a shaped array interpreted as a RGB image can be accessed as follows: 

          red   = img.get(0,x,y);
      green = img.get(1,x,y);
      blue  = img.get(2,x,y);
    where img is an Array3D object and (x,y) are the pixel coordinates.

    RGBA images

    A RGBA image is stored into a shaped array as a 3D array with the first dimension equals to 4, the second dimension equals to the width of the image and the third dimension equals to the height of the image. The pixels of a RGBA image have four values: red (R), blue (B), green (G) and alpha (A). The values of a shaped array interpreted as a RGBA image can be accessed as follows: 

          red   = img.get(0,x,y);
      green = img.get(1,x,y);
      blue  = img.get(2,x,y);
      alpha = img.get(3,x,y);
    where img is an Array3D object and (x,y) are the pixel coordinates.

    Currently, to avoid ambiguities about the interpretation of the alpha channel, we only support RGBA images with byte values. These values are understood as being unsigned. A value of zero for alpha corresponds to a transparent pixel, the highest possible value of alpha, i.e. 0xFF, corresponds to an opaque pixel. It is assumed by the various conversion routines, that the color levels are not pre-multiplied by the alpha value. When converting a buffered image of type BufferedImage.TYPE_INT_ARGB_PRE or BufferedImage.TYPE_4BYTE_ABGR, the pre-multiplication is reversed to follow this convention.
