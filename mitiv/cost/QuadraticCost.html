    Implementation of a quadratic cost function. A general form of a quadratic cost is: 
         f(x) = (H.x - y)'.W.(H.x - y)
    where H and W are linear operators and y is a vector of the the output space of H.

      • QuadraticCost

        public QuadraticCost(LinearOperator H,
             Vector y,
             LinearOperator W)
        Constructor for a general quadratic cost function. Create an instance of the differentiable cost function: 
             f(x) = (H.x - y)'.W.(H.x - y)
        where H and W are linear operators and y is a vector of the the output space of H. Operator W must map the output space of H to the output space of H. Operator W should be symmetric and positive definite (respectively semi-definite) for f(x) to be strictly convex (respectively convex).

        Having any of these arguments equal to null amounts to ignore them in the formula. Thats is, y = null is the same (but faster) as having y a vector of zeros; H = null (or W = null) is the same (but faster) as assuming that H (or of W) is the identity. Though not all the arguments can be null as at least one of these is needed to determine the input space of f(x).

        Note that none of the components H, y, nor W are copied. If the caller does change one of these while using the cost function, the cost function will use the actual contents of the components. If this is inappropriate, the cost must be instantiated with copies of the components which are subject to changes.

        Parameters:
        H - - A linear operator.
        y - - A vector of the output space of H.
        W - - An endomorphism of the output space of H..

      • computeCostAndGradient

        public double computeCostAndGradient(double alpha,
                            Vector x,
                            Vector gx,
                            boolean clr)
        Description copied from interface: DifferentiableCostFunction
        Compute the value of the cost function and its gradient.

        This method is intended to compute the value and the gradient of a cost function which are needed by limited memory non-linear optimization methods like the non-linear conjugate gradient and quasi-Newton methods such as L-BFGS.

        The rationale of the alpha and clear arguments is to let one builds a composite cost function efficiently. Below is the pseudo-code of an example which shows how to compute the value and the gradient of a function which is a linear combination of other functions (in the array f) with different multipliers (in the array alpha): 

             DifferentiableCostFunction[] f;
     double[] alpha;
     Vector x;
     Vector gx;
     double fx;
     fx = 0.0;
     for (int j = 0; j < f.length; ++j) {
         fx += f[j].computeCostAndGradient(alpha[j], x, gx, (j == 0));
     }
        Note that the gradient vector has to be cleared for the first (j == 0) function of the list.
        Specified by:
        computeCostAndGradient in interface DifferentiableCostFunction
        Parameters:
        alpha - - A non-negative multiplier for the cost.
        x - - The vector of variables.
        gx - - The vector to store the gradient of the cost function times the weight.
        clr - - Indicate whether the gradient vector has to be cleared (that is, filled with zero) prior to the computation. If false, the contents of the gradient vector is incremented with the gradient of the cost function.
        Returns:
        The value of the cost function times the weight.
