Interface ProximalOperator

  • All Superinterfaces:

    All Known Implementing Classes:

    public interface ProximalOperator
extends CostFunction
    Interface for cost functions implementing their proximal operator. The proximal operator of a function f : R^n -> R^n is defined by: 
         prox_f(x) = argmin_y { f(y) + (1/2) ||x - y||^2 }
    Functions for which the proximal operator can be computed can be minimized even though there are non-smooth and thus non-differentiable. They are notably useful to implement constraints or to impose sparsity. See reference below for a review.
    1. Parikh, N. & Boyd, S. "Proximal Algorithms," in Foundations and Trends in Optimization, Vol. 1, pp. 123-231 (2013) web doi.

    Éric Thiébaut

      • applyProx

        void applyProx(double alpha,
             Vector inp,
             Vector out,
             double tol)
        Apply the proximal operator of a cost function. The purpose of this method is to find the minimum of: 
         alpha*f(out) + (1/2)*norm2(out - inp)
        with respect to vector out and given weight alpha and the input vector out. The argument tol is a relative tolerance parameter to find an approximate solution to the problem.
        Parameters:
        alpha - - A non-negative weight for the cost.
        inp - - The vector of input variables.
        out - - The vector of output variables.
        tol - - A non-negative tolerance parameter.
