mitiv.base.indexing

Class Range

  • java.lang.Object
    • mitiv.base.indexing.Range

  • public class Range
extends java.lang.Object
    Manage ranges along array dimensions.

    Symbolically, ranges are specified as [start:stop:step] or just as [start:stop] (the step is optional). As in Java, indexes of a range start at 0 (also FIRST) but may be negative. Starting/ending indexes of a range are interpreted as offsets relative to the length of the dimension of interest if they are strictly negative. That is -1 (also LAST) is the last element along a dimension, -2 is the penultimate element, etc. This rule is however only applied once: assuming n is the length of the dimension of interest and j is the first or last index, then j (if j ≥ 0) or j + n (if j < 0) must be greater or equal 0 and strictly less than n.

    The convention is that the element corresponding to the first index of a range is always considered (unless the range is empty as explianed below) while the last one may not be reached. In pseudo-code, the rules are: 

         if (step > 0) {
         for (int index = first; index <= last; index += step) {
             ...
         }
     } else if (step < 0) {
         for (int index = first; index >= last; index += step) {
             ...
         }
     } else {
         // step = 0 is illegal
         throw new IllegalRangeException();
     }
    Note that a range is empty if first > last and step > 0 or if first < last and step < 0 (after applying the rules for negative indices).
