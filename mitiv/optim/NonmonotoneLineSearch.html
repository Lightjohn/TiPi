    Nonmonotone line search method.

    This calls implements nonmonotone line search method of Birgin, Martinez, and Raydan (2000).

    1. Ernesto G. Birgin, José Mario Martínez, and Marcos Raydan. " Nonmonotone spectral projected gradient methods on convex sets." SIAM J. Optim., 10, pp. 1196–1211, (2000). [doi: 10.1137/S1052623497330963]
    2. Ernesto G Birgin, José Mario Martínez, and Marcos Raydan. "Algorithm 813: SPG — software for convex-constrained optimization." ACM Transactions on Mathematical Software (TOMS), 27, pp. 340–349 (2001). [doi: 10.1145/502800]
Author:
Éric Thiébaut.

      • setTolerance

        public void setTolerance(double ftol)
        Set the parameter for the Armijo-like condition.

        The line search is assumed to have converged when the following sufficient decrease (Armijo) condition holds: 

            f(x0 + alpha*d) <= fmax + alpha*ftol*
        where fmax is the highest function value since the last m steps, alpha is the step length and ftol is the parameter of the Armijo condition.

        Parameters:
        ftol - - The parameter value. Must be in the range (0,1) otherwise an IllegalArgumentException is thrown.

      • startHook

        protected void startHook()
        Description copied from class: LineSearch
        Protected method to set internals at the start of a line search. This protected method is called by the LineSearch.start(double, double, double, double, double) method to set attributes of the line search instance after starting a new search. It can be overwritten as needed, the default method does nothing but set the next task to be LineSearch.SEARCH. Derived line search classes which override this method should call @{link success} or @{link failure} to set the next task on success or to set the reason of the error on failure.

      • iterateHook

        public void iterateHook(double f,
               double g)
        Description copied from class: LineSearch
        Protected abstract method to iterate during a line search. This protected method is called by the LineSearch.iterate(double, double, double) method to check whether line search has converged and, otherwise, to compute the next step to try (stored as attribute stp). The provided arguments have been checked. Upon return, the caller method, LineSearch.iterate(double, double, double), takes care of safeguarding the step. This method in derived line search classes must call @{link success} or @{link failure} to set the next task on success or to set the reason of the error on failure.
        Specified by:
        iterateHook in class LineSearch
        Parameters:
        f - The value of the function at x = x0 + stp*p where x0 are the variables at the start of the line search and p is the search direction.
        g - The directional derivative at x, that is p'.g(x) the inner product between the search direction and the function gradient at x.
